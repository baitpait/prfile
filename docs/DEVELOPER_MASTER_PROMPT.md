# برومبت المبرمج — مشروع «بروفايل ميدا» (Profile Media)

انسخ المحتوى التالي كما هو (أو ضمن تذكرة Jira/Linear) للمبرمج أو لفريق التنفيذ.

---

## 1) الهوية والهدف والمكدس
أنت مطوّر/فريق تنفيذ لمشروع **بروفايل ميدا**: منصة **تقارير وواجهات شاملة** لإدارة أعمال شركة إعلامية/هوية رقمية.

- **الباكند (معتمد):** **Laravel** على **PHP 8.2+**.  
- **الفرونت (افتراضي المشروع):** **Blade + Livewire 3 + Alpine.js + Tailwind CSS** (RTL، واجهة عربية فقط).  
- **بديل الفرونت (عند الحاجة):** **Inertia.js + Vue 3** — يتطلب توثيقاً في ADR إذا أصبح المسار الأساسي للمشروع.

المرجع: `docs/decisions/ADR-001-backend-laravel-frontend-stack.md` و`docs/06_RECOMMENDED_LANGUAGES_AR.md`.

## 2) الالتزام بالدستور (إلزام — لا يُسقط)

- **الدستور الكامل:** `profile-mida/.cursorrules` — اقرأه **كاملاً** قبل أول دمج إلى الفرع الرئيسي.  
- **واجهة عربية فقط (RTL) + الاستثناءات:** `profile-mida/docs/00_CONSTITUTION_UI_AR.md` (نسخة منقولة من §0bis).  
- **أي كود أو واجهة تخالف `.cursorrules` أو `00_CONSTITUTION_UI_AR.md` يُعاد حتى التصحيح** — يُعتبر المخرج غير مقبول للمراجعة.  
- الهوية البصرية: `profile-mida/docs/05_VISUAL_IDENTITY_AR.md`.  
- المكدس واللغات: `profile-mida/docs/06_RECOMMENDED_LANGUAGES_AR.md` و`profile-mida/docs/decisions/ADR-001-backend-laravel-frontend-stack.md`.

## 2bis) مصدر الحقيقة للنطاق والتقارير والبيانات

## 3) النطاق الوظيفي (MVP)
1. مستخدمون + أدوار + تسجيل دخول آمن.  
2. عملاء + موردون + جهات اتصال.  
3. فواتير + بنود **نص حر** (بدون كتالوج منتجات إجباري).  
4. أوامر شراء + بنود نص.  
5. **دفعات** مرتبطة بـ **العميل أو المورد** (ليس بالفاتورة): مبلغ، عملة، `paid_at`، طريقة، مرجع.  
6. مصروفات وإيرادات **نصية** (وصف مركزي).  
7. **تقارير:** كشف حساب **متعدد العملة** (قسم لكل عملة)، لوحة تنفيذية، تصدير CSV/Excel + PDF للكشوف.  
8. (اختياري لاحقاً) استيراد من SQLite مرجعي إن وُجد في المستودع — لا تفترض وجوده في كل بيئة.

## 4) غير المطلوب في المرحلة الأولى
- إدخال SPA منفصلة بالكامل (React/Next فقط) **بدون ADR** يبرر التعقيد التشغيلي.  
- تطبيق جوال أصلي (يمكن تأجيله).  
- كتالوج منتجات كمجال إلزامي للفوترة.

## 5) المتطلبات غير الوظيفية
- **الأداء:** pagination؛ فهارس على FK والتقارير؛ تجنب N+1.  
- **الأمان:** لا أسرار في Git؛ تشفير كلمات المرور؛ سياسة سجلات بدون PII.  
- **الوصول:** تباين مقبول وتسميات واضحة للتقارير المطبوعة.  
- **الاختبارات:** مسارات سعيدة + فاشلة للصلاحيات والتقارير المالية.

## 6) مخرجات مطلوبة من أول سباق تنفيذ (Sprint 0)
1. مستودع Git مع README يشرح التشغيل المحلي (**Laravel Sail أو Docker أو `composer` محلي** — وثّق ما اخترت).  
2. الالتزام بـ **`docs/decisions/ADR-001-backend-laravel-frontend-stack.md`**؛ أي تغيير جذري للفرونت يُحدَّث في ADR جديد.  
3. مخطط قاعدة (Laravel migrations) يطابق `03_DATABASE_SPEC.md` مع soft delete حيث ينطبق المخطط.  
4. شاشة واحدة على الأقل للتقارير: **كشف حساب عميل متعدد العملة** (Blade/Livewire أو Inertia حسب المسار المعتمد).  
5. قالب PDF بسيط لكشف حساب (Laravel + مكتبة PDF معتمدة في المشروع).

## 6bis) لغة الكود والواجهة واللغات البرمجية
- الكود وأسماء الرموز: **إنجليزي**.  
- **واجهة المستخدم: عربية فقط (RTL)** — ملزم بالدستور §0bis و`docs/00_CONSTITUTION_UI_AR.md`؛ الهوية من `docs/05_VISUAL_IDENTITY_AR.md`.  
- استثناءات العرض: رموز العملات ISO والأرقام داخل خلايا `dir="ltr"` عند الحاجة.  
- **المكدس:** **PHP/Laravel** للباكند؛ **Blade + Livewire + Alpine + Tailwind** للفرونت الافتراضي؛ **SQL** عبر migrations؛ **PHPUnit/Pest**؛ تفاصيل إضافية في **`docs/06_RECOMMENDED_LANGUAGES_AR.md`**.  
- أي انحراف عن مسار الفرونت الافتراضي → ADR.

## 7) معايير القبول (Acceptance)
- [ ] إنشاء عميل + فاتورتان بعملتين مختلفتين + دفعتان بنفس عملتَي الفاتورة → كشف الحساب يظهر **رصيدين منفصلين** صحيحين.  
- [ ] لا يمكن للمستخدم «قراءة فقط» تعديل دفعة.  
- [ ] حذف منطقي لا يزيل السجل من تقارير «الأرشيف» إن وُجدت سياسة عرض.  
- [ ] تصدير CSV يعيد نفس الأرقام الظاهرة في الشاشة.

## 8) ما يجب أن تسأل عنه صاحب المشروع قبل البناء المعمّق
- سياسة **دفعة بعملة مختلفة عن الفاتورة** (ممنوع / مسموح بتسوية يدوية فقط؟).  
- الهوية البصرية (شعار، ألوان) للتقارير المطبوعة.  
- بيئات الاستضافة المفضّلة (سحابة/محلي).
