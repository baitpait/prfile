# قاعدة البيانات — مجلد مرجعي (بروفايل ميدا)

## الغرض
هذا المجلد يجمع **المرجعية** لمخطط البيانات والترحيل. **لم يُحسم** محرك التخزين النهائي (PostgreSQL / SQLite / غيره) في الدستور؛ يُحدَّد في `docs/decisions/ADR-001-*.md`.

## ملفات جاهزة الآن
| الملف | الوصف |
|--------|--------|
| **`business_v1.sqlite`** | قاعدة SQLite **مُرحَّلة** من النسخة الاحتياطية (عملاء، فواتير، بنود، موردون، أوامر شراء، مصروفات، إيرادات، أرشيف كتالوج قديم، `source_row_json` للتدقيق). |
| **`reference_sqlite_v1_schema.sql`** | تعريف الجداول (للمراجعة أو إعادة الإنشاء). |

## محتويات مقترحة (بعد اختيار التقنية)
- `migrations/` — هجرات قابلة للعكس حيث أمكن.  
- `seed/` — بيانات بذرة (مستخدم إداري، عملة افتراضية، إلخ).  
- `erd/` — مخطط ERD (PNG أو dbdiagram.io) مُحدَّث مع كل تغيير هيكلي.

## مرجع ترحيل قائم (خارج هذا المجلد اختيارياً)
في نفس سطح مجلدات سطح المكتب/المشروع قد يوجد:
- `database/business_v1.sqlite` — قاعدة مُرحَّلة من XML كمرجع للبيانات التاريخية.  
- سكربتات الترحيل بجانبها.

عند بناء المنتج النهائي: إما **استيراد** من ذلك الملف أو **إبقاؤه أرشيفاً** فقط — يُوثَّق القرار في ADR.

## قواعد التعديل
- أي تغيير على أسماء الحقول المستخدمة من الواجهة → تحديث `docs/03_DATABASE_SPEC.md` + إشعار الفرونت.  
- لا حذف فيزيائي لبيانات حرجة بدون سياسة موثّقة.
