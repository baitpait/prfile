انسخ النص من **«ابدأ من هنا»** حتى **«نهاية البرومبت»** في رسالة واحدة للمبرمج.

## ابدأ من هنا

أنت تبدأ العمل على مشروع **بروفايل ميدا (Profile Media)** — منصة داخلية لـ **تقارير وواجهات شاملة** (عملاء، موردون، فواتير، أوامر شراء، دفعات، مصروفات/إيرادات، صلاحيات). المجلد الذي بين يديك يحتوي الدستور والوثائق وجاهز للاندماج في مستودع Laravel.

### 1) المكدس (ملزم)
- **Backend:** **Laravel**، **PHP 8.2+**.
- **Frontend الافتراضي:** **Blade + Livewire 3 + Alpine.js + Tailwind CSS** مع **RTL** كامل.
- **بديل مسموح بتوثيق ADR:** **Inertia.js + Vue 3** إذا قررتم أنه المسار الأساسي للمشروع.
- **قاعدة البيانات:** **SQL** عبر Laravel migrations؛ الإنتاج يفضّل MySQL أو PostgreSQL؛ للتطوير يُقبل SQLite.
- **اختبارات:** **Pest** أو **PHPUnit**.
- التفاصيل: `docs/decisions/ADR-001-backend-laravel-frontend-stack.md` و`docs/06_RECOMMENDED_LANGUAGES_AR.md`.

### 2) الدستور (اقرأه قبل أول دمج)
- **كامل:** `.cursorrules` (إصدار 1.1).
- **واجهة عربية فقط:** `docs/00_CONSTITUTION_UI_AR.md` — لا عناوين ولا قوائم تنقل إنجليزية في الإنتاج؛ **استثناء:** رموز العملات ISO (`ILS`, `USD`, …) وأرقام المبالغ في خلايا `dir="ltr"` عند الحاجة.
- **الهوية البصرية:** `docs/05_VISUAL_IDENTITY_AR.md` + الشعار `branding/logo-official.png` (استخرج ألوان HEX النهائية من الملف عند بناء الثيم).
- **مخالفة الدستور = رفض المراجعة** حتى التصحيح.

### 3) نطاق المنتج (منطق العمل — ملخص الدردشة)
- **شركة خدمات:** لا كتالوج منتجات/خدمات إلزامي للفوترة؛ **بنود الفاتورة وأمر الشراء نص حر** (عنوان + وصف + أسعار + كمية).
- **الدفعات:** جداول دفعات مرتبطة بـ **العميل** أو **المورد** — **ليس** مرتبطة بالفاتورة في النموذج الجديد؛ كل دفعة: مبلغ، **عملة خاصة بها**، **`paid_at` (UTC في التخزين)**، طريقة، مرجع، ملاحظات.
- **التقارير المالية:** **كشف حساب متعدد العملة** — **قسم لكل عملة**؛ لا جمع دولار + شيكل في رقم واحد بدون سعر صرف موثّق وسياسة.
- **المصروفات:** حقل **وصف نصي مركزي** (ليس نموذج مورد/تصنيف إجباري من جداول مرجعية).
- **الإيرادات العامة:** سطر إيراد نصي عندما لا يكون «دفعة عميل».
- **الحذف:** soft delete للبيانات الحرجة حيث ينطبق المخطط.
- **الترحيل (مرجع):** يوجد `database/business_v1.sqlite` (بيانات مُرحَّلة من نظام XML قديم) و`database/reference_sqlite_v1_schema.sql` — صفوف كثيرة تحتوي `source_row_json` لأغراض التدقيق؛ **لا تفترض** أن كل منطق الأعمال القديم مُعاد نمذجته في Laravel بعد — استخدمها كمرجع/استيراد لاحقاً حسب تذكرة منفصلة.

### 4) وثائق يجب قراءتها كاملة (بالترتيب)
1. `docs/01_VISION_AND_SCOPE_AR.md`  
2. `docs/03_DATABASE_SPEC.md`  
3. `docs/04_REPORTS_AND_UI_MATRIX.md`  
4. `docs/DEVELOPER_MASTER_PROMPT.md` (المعايير التفصيلية ومعايير القبول)

### 5) Sprint 0 — مخرجات مطلوبة
1. مشروع Laravel قابل للتشغيل محلياً + README (Sail/Docker أو composer — وثّق).  
2. Migrations أولية تعكس `03_DATABASE_SPEC.md` (مع soft delete حيث ينطبق).  
3. **شاشة واحدة على الأقل:** كشف حساب عميل **متعدد العملة** (واجهة عربية RTL).  
4. **قالب PDF** بسيط لكشف حساب (عربي + شعار عند التوفر).  
5. فروع Git بميزات؛ Conventional Commits.

### 6) أسئلة عالقة — اسأل صاحب المشروع قبل التعمق
- سياسة **دفعة بعملة مختلفة عن فاتورة** العميل (ممنوع / مسموح بتسوية يدوية فقط؟).  
- بيئة الاستضافة المستهدفة.  
- تأكيد الألوان النهائية من الشعار.

### 7) معايير قبول سريعة (لا تغلق Sprint 0 بدونها)
- عميل + فاتورتان بعملتين مختلفتين + دفعتان بنفس عملتي الفاتورة → الكشف يظهر **رصيدين منفصلين** صحيحين.  
- مستخدم «قراءة فقط» لا يعدّل دفعة.  
- تصدير CSV يطابق الأرقام المعروضة.
