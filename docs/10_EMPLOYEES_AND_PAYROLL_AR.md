# الموظفون والرواتب — بروفايل ميدا

## الغرض
وحدة **مستقلة** لإدارة بيانات الموظفين وسجل الرواتب الشهرية، منفصلة عن:
- جدول `expenses` (المصروفات)
- جدول `users` (لا يوجد `user_id` على الموظف)

## الصلاحيات
| الإجراء | مدير | محاسب | مشاهد |
|---------|------|-------|-------|
| عرض الموظفين والرواتب | ✓ | ✓ | ✓ |
| إضافة/تعديل/حذف موظف | ✓ | — | — |
| تسجيل/تعديل/حذف راتب | ✓ | ✓ | — |
| تقرير الرواتب (عرض) | ✓ | ✓ | ✓ |
| تصدير PDF/CSV للرواتب | ✓ | ✓ | — |

## المسارات
- `/employees` — قائمة الموظفين
- `/employees/create` — إضافة موظف (مدير)
- `/employees/{id}` — ملف الموظف + سجل رواتبه
- `/salary-payments` — سجل الرواتب
- `/salary-payments/create` — تسجيل راتب (محاسب+)
- `/reports/salaries` — تقرير الرواتب حسب فترة الشهر

## الجداول
### `employees`
بيانات الموظف: الاسم، الرقم الوظيفي، التواصل، القسم، الراتب الأساسي وعملته، البنك، الحالة.
`recorded_by_user_id` يُسجّل من أضاف السجل (للتدقيق فقط).

### `salary_payments`
راتب شهر لموظف: سنة/شهر، أساسي، مكافأة، خصم، صافي، عملة، حالة (`draft` / `paid` / `cancelled`).
**قيد فريد:** `(employee_id, period_year, period_month, currency_code)`.

## قواعد العمل
1. **لا دمج عملات** في مجاميع التقرير — كل عملة تُعرض منفصلة.
2. الصافي = أساسي + مكافأة − خصم (لا يقل عن صفر).
3. عند الحالة `paid` يُلزم تاريخ الدفع وطريقة الدفع.
4. الحذف **ناعم** (`soft deletes`) على الموظف والراتب.

## الربط بالتقارير المالية
تقرير الرواتب **لا يُضاف** تلقائياً إلى المصروفات أو التدفق النقدي. أي دمج مستقبلي يتطلب قراراً صريحاً ووحدة منفصلة.
