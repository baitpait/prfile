# مبادئ هندسية — بروفايل ميدا (ملخص يكمّل `.cursorrules`)

## 1) الطبقات
- **عرض (UI):** عرض وقراءة وتصدير؛ لا منطق أعمال معقّد دون طبقة خدمة.
- **خدمات المجال:** قواعد العمل، التحقق، تجميع التقارير.
- **وصول للبيانات:** مستودع/استعلامات؛ فهارس على المفاتيح الأجنبية والتقارير الثقيلة.

## 2) التقارير
- كل تقرير له: **تعريف العملة**، **نطاق زمني**، **فلتر صلاحيات**، **تعريف الحالات** (مستبعد void، محذوف منطقياً، …).
- لا «إجمالي واحد» يخلط عملتين؛ إن لزم «إجمالي إداري» يُعرض مع **سعر صرف وتاريخ** موثّقين.

## 3) الصلاحيات
- نموذج أدوار (مثلاً: مدير، محاسب، تنفيذ، قراءة فقط) — يُفصّل في مرحلة التنفيذ.
- كل استعلام تقرير يفرض **نطاق المستخدم** (عميل/مشروع/فرع).

## 4) الأداء
- ترقيم الصفحات للقوائم الطويلة؛ تجنب N+1؛ تخزين مؤقت للوحات الثقيلة عند الحاجة مع سياسة إبطال.

## 5) الملاحظات التشغيلية
- بيئات: Dev / Staging / Prod بنفس شكل المخطط قدر الإمكان.
- مراقبة: أخطاء حرجة مع سياق تقني **دون** PII.
