# رؤية ونطاق — بروفايل ميدا

## الرؤية
بناء منصة داخلية/شبه داخلية لشركة إعلامية تعمل تحت اسم **بروفايل ميدا**، تركّز على **رؤية شاملة للعمل** عبر **تقارير موثوقة** و**واجهات تغطي دورة العمل** من أول اتصال بالعميل حتى التسليم والمتابعة المالية عند الحاجة.

## اللغة
- **واجهة المستخدم: عربية فقط (RTL).** التفاصيل في `docs/05_VISUAL_IDENTITY_AR.md` و`.cursorrules`.

## النطاق (MVP مقترح)
1. **إدارة العملاء والموردين** (بيانات أساسية، جهات اتصال، ملاحظات).
2. **المشاريع والتسليمات** (مراحل، تواريخ، مسؤول، حالة).
3. **المستندات التجارية** (فواتير/عروض — بنود **نص حر** دون كتالوج إجباري).
4. **الدفعات على مستوى الطرف** (عميل/مورد) مع عملة وتاريخ لكل دفعة — لا دمج أعمى بين العملات.
5. **المصروفات والإيرادات النصية** (وصف حر + مبلغ + عملة + تاريخ).
6. **التقارير والواجهات:** لوحات تنفيذية، كشوف حساب متعددة العملة، تقارير مشاريع، تصدير للطباعة/Excel.

## خارج النطاق (حالياً)
- اختيار **إطار باكند أو فرونت** نهائي — يُؤجل حتى قرار ADR صريح.
- تطبيقات جوال أصلية (قد تُعاد النظر لاحقاً كمرحلة).

## مؤشرات النجاح
- المستخدم يجد **رقماً واحداً صحيحاً** لرصيد العميل **لكل عملة** دون التباس.
- أي صف مستورد من بيانات قديمة يبقى **قابلاً للتدقيق** عبر أرشيف JSON أو مرجع ترحيل.
